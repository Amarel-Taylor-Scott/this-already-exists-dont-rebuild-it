# Agent instructions

- Search for an existing package, component, extractor, index, or benchmark before implementing a
  replacement.
- Use plain names: package release, operation, representation, label, blocking key, adapter,
  compatibility claim, route, execution record, and receipt.
- Do not equate catalog entries with verified reusable capabilities.
- Do not execute untrusted package code in the ingestion process.
- Do not convert inferred labels or model output into facts without evidence and review state.
- Preserve failures and negative evidence.
- Make profiles and limits explicit configuration records.
- Never report token, cost, accuracy, scale, or acceptance claims without a generated receipt.
